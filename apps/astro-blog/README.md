# Astro Blog

`estrivault-blog-app` の描画層を Astro で提供するアプリです。

## Commands

```bash
pnpm --filter astro-blog dev
pnpm --filter astro-blog build
pnpm --filter astro-blog preview
pnpm --filter astro-blog check
pnpm --filter astro-blog test:e2e
```

## Routes

- `/`
- `/[page]/`
- `/post/[slug]`
- `/category/[category]/`
- `/category/[category]/[page]/`
- `/tag/[tag]/`
- `/tag/[tag]/[page]/`
- `/sitemap.xml`
- `/llms.txt`
- `/llms-full.txt`

## URL Canonicalization and Redirects

Canonical URLs follow these rules:

- Document pages omit the trailing slash: `/post/[slug]`
- Archive pages keep the trailing slash: `/category/[category]/`, `/tag/[tag]/`
- Archive page 1 omits the page number; page 2 and later include it: `/category/[category]/2/`

Redirects to canonical URLs are generated by `scripts/generate-redirects.mjs` as
`public/_redirects` before `astro build`, then copied to `dist/_redirects` during the Astro static
build. This uses Cloudflare Workers Static Assets `_redirects` support to return HTTP 301 redirects.
The generated `public/_redirects` file is ignored by git; update the script instead of editing the
generated file.

This is intentionally Cloudflare-specific. Astro's static redirects are not used for this URL
canonicalization because they do not cover hosting-level trailing slash redirects as HTTP 301
responses in a static deployment. If the site is moved away from Cloudflare, port these redirect
rules to the new host or replace them with an adapter/server-side redirect layer.
